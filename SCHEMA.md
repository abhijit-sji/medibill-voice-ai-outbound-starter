## 1. `call_logs`

## 2. `cost_logs`

Tracks per-call API usage costs for all external service providers. For outbound ElevenLabs calls, `service = 'elevenlabs'` and `total_cost` reflects ElevenLabs conversation charges (character/minute-based pricing).

### Columns

| Column         | Type          | Nullable | Default  | Description                                                                              |
|---------------|---------------|----------|----------|------------------------------------------------------------------------------------------|
| `id`          | UUID          | no       | `uuid_generate_v4()` | Primary key                                                              |
| `clinic_id`   | UUID          | yes      | â€”        | FK â†’ `clinics.id` (CASCADE delete)                                                       |
| `call_log_id` | UUID          | yes      | â€”        | FK â†’ `call_logs.id` (SET NULL on delete)                                                 |
| `service`     | TEXT          | no       | â€”        | Provider: `openai`, `twilio`, `elevenlabs`, `stripe`, `sendgrid`, or `resend`            |
| `cost_type`   | TEXT          | no       | â€”        | Category: `api_call`, `voice_minutes`, `sms`, `email`, or `transaction_fee`              |
| `amount`      | DECIMAL(10,4) | no       | â€”        | Cost amount in `currency`                                                                |
| `currency`    | TEXT          | yes      | `'USD'`  | ISO 4217 currency code                                                                   |
| `units`       | DECIMAL(10,2) | yes      | â€”        | Quantity consumed (tokens, seconds, messages, transactions)                              |

## 3. `ai_interactions`

Stores a granular, turn-by-turn log of every exchange in a call. Each row represents one utterance from the AI agent or the patient, plus any function calls executed during that turn. For ElevenLabs outbound calls, rows are written by `elevenlabs-webhook` when `conversation.message` events arrive.

### Columns

| Column               | Type          | Nullable | Default  | Description                                                                      |
|--------------------|---------------|----------|----------|----------------------------------------------------------------------------------|
| `id`               | UUID          | no       | `uuid_generate_v4()` | Primary key                                                      |
| `call_log_id`      | UUID          | yes      | â€”        | FK â†’ `call_logs.id` (CASCADE delete)                                             |
| `sequence_number`  | INTEGER       | no       | â€”        | Turn order within the call (1-indexed)                                           |
| `speaker`          | TEXT          | no       | â€”        | `ai`, `patient`, or `system`                                                     |
| `message`          | TEXT          | no       | â€”        | Raw text of the utterance                                                        |
| `intent`           | TEXT          | yes      | â€”        | Classified intent: `greeting`, `schedule`, `provide_info`, `confirm`, `farewell` |
| `entities_extracted`| JSONB        | yes      | â€”        | Extracted named entities: `{ name, dob, phone, provider, date, time }`           |
| `confidence`       | DECIMAL(3,2)  | yes      | â€”        | Speech recognition or intent confidence (0.00â€“1.00)                              |
| `function_called`  | TEXT          | yes      | â€”        | Function name: `check_availability`, `book_appointment`, `find_patient`, etc.    |
| `function_args`    | JSONB         | yes      | â€”        | Arguments passed to the function call                                            |
| `function_result`  | JSONB         | yes      | â€”        | Return value from the function call                                              |
| `timestamp`        | TIMESTAMPTZ   | yes      | `NOW()`  | Time this turn occurred                                                          |

### Column Name Note

The `call_log_id` column links `ai_interactions` to `call_logs`. The content of each turn is in the `message` column â€” not `input_text` or `output_text`. Always use `call_log_id` and `message` when querying.

## Row Level Security Summary

All four tables have RLS enabled. The `SUPABASE_SERVICE_ROLE_KEY` bypasses all policies (used exclusively by edge functions). Authenticated clinic users can read their own clinic's data.

| Table                       | Read                            | Write                 |
|----------------------------|---------------------------------|-----------------------|
| `call_logs`                | Service role + clinic users     | Service role only     |
| `cost_logs`                | Service role + clinic users     | Service role only     |
| `ai_interactions`          | Service role + clinic users     | Service role only     |
| `ai_agent_configurations`  | Clinic users (own clinic)       | Admins only           |
